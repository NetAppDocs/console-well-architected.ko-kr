[role="lead"]
사용자 지정 규칙을 사용하면 Well-architected 대시보드에서 NetApp 모범 사례와 함께 자체 아키텍처 표준을 설정, 확인 및 적용할 수 있습니다.



[[overview]]
== 개요

규칙 작성기를 사용하면 일반적인 언어로 사용자 지정 모범 사례 규칙을 만들 수 있습니다. 이러한 규칙은 모든 리소스에 대해 검사됩니다.

사용자 지정 규칙을 사용하면 일반적인 공급업체 규칙으로는 충분히 다룰 수 없는 조직의 표준, 위험 한도 및 운영 요구 사항을 시행할 수 있습니다.



[[how-custom-rules-work]]
== 사용자 지정 규칙 작동 방식

사용자 지정 규칙은 AI 기반 프로세스를 사용하여 일반 언어로 작성된 지침을 명확하고 구조화된 규칙으로 변환합니다. AI는 번역가 및 검토자 역할을 합니다. 규칙을 제안하고 요청을 어떻게 이해했는지 설명한 다음, 규칙을 저장하거나 예약하기 전에 사용자의 승인을 요청합니다.

이 과정은 다음을 보장합니다.

* 규칙 논리는 명확하게 정의되어 있으며 오류 검사를 거칩니다.
* 자유 형식 코드는 실행되지 않습니다.
* 규칙을 승인하기 전에 어떤 항목이 평가될지 정확히 확인할 수 있습니다.




[[before-you-begin]]
== 시작하기 전에

사용자 지정 규칙을 만들고 실행하기 전에 다음 사항을 확인하십시오.

AWS 자격 증명:: AWS 환경에 대한 읽기 전용 액세스 권한입니다. 사용자 지정 규칙은 보안 경계로 읽기 전용 권한을 요구합니다. link:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["Workload Factory에 AWS 자격 증명 추가"^].
ONTAP 자격 증명:: 읽기 전용 권한이 있는 FSx 뷰어 사용자입니다. Workload Factory는 필요한 권한을 부여하여 이 사용자를 자동으로 생성합니다.
링크 또는 에이전트:: 규칙 평가 시 FSx 파일 시스템에 연결하는 데 필요합니다. link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["링크를 사용하여 FSx for ONTAP 파일 시스템에 연결합니다."^].




[[limitations]]
== 제한 사항

사용자 지정 규칙에는 다음과 같은 제한 사항이 적용됩니다.

* 자동 수정 기능은 제공되지 않습니다. 제공된 지침을 사용하여 규정을 준수하지 않는 리소스를 직접 수정해야 합니다.
* 각 규칙은 한 가지 유형의 리소스만 확인할 수 있습니다. 여러 유형의 리소스를 확인하려면 별도의 규칙을 만드세요.
* 과도한 스캔을 방지하기 위해 최소 평가 간격은 1시간입니다.
* 규칙은 읽기 전용 액세스 권한을 가지며 환경을 변경할 수 없습니다.




[[rule-definition-model]]
== 규칙 정의 모델

각 사용자 지정 규칙은 다음과 같은 구성 요소로 이루어져 있습니다.

규칙 이름:: 규칙에 대한 사용자 정의 이름입니다.
설명:: 이 규칙이 조직에 중요한 이유에 대한 근거입니다.
리소스 유형:: 평가할 FSx 리소스(파일 시스템, 볼륨 또는 캐시). 현재 규칙당 하나의 리소스 유형만 허용됩니다.
스케줄러:: 규칙이 평가되는 빈도(최소 1시간, 기본값 24시간).
조건 논리:: 확인해야 할 구체적인 기준.
평가 범위:: 평가 대상 리소스(범위 내 모든 해당 리소스).
교정 논리:: 규정을 준수하지 않는 리소스를 처리하기 위한 수동 지침입니다.




[[rule-lifecycle]]
== 규칙 수명 주기

사용자 지정 규칙은 다음 단계를 거쳐 진행됩니다.

초안:: 규칙이 생성되고 있습니다. 평가는 수행되지 않습니다.
드라이런:: 어떤 리소스가 영향을 받을지 미리 확인하기 위한 일회성 평가입니다. 일정 수립 전 유효성 검사에 사용됩니다.
예약됨(활성화됨):: 규칙이 활성화되어 있으며 정의된 일정에 따라 실행됩니다.
비활성화됨:: 규칙이 일시 중지되었지만 삭제된 것은 아닙니다. 다시 활성화할 수 있습니다.
삭제됨:: 해당 규칙은 영구적으로 삭제되었습니다.




[[custom-rule-examples]]
== 사용자 지정 규칙 예시

다음 예시는 일반적인 사용자 지정 규칙을 보여줍니다.

*프로덕션 볼륨 보호*:: "'Prod' 태그가 있는 모든 볼륨에는 스냅샷 정책이 설치되어 있어야 합니다."
+
--
이 규칙은 회사 요구 사항에 따라 프로덕션 볼륨을 보호하기 위해 자동으로 스냅샷을 생성합니다.

--
*아카이브 볼륨 계층화*:: "'Archive' 태그가 있는 모든 볼륨에는 계층화 정책이 'All'로 설정되어야 합니다."
+
--
이 규칙은 보관된 데이터를 더 저렴한 스토리지로 자동으로 이동하여 비용을 절감합니다.

--
*파일 시스템 명명 규칙*:: "모든 파일 시스템은 명명 규칙 'fsx-{environment}-{application}'을 따라야 합니다."
+
--
이 규칙은 파일 시스템이 조직의 명명 규칙을 준수하여 관리 및 비용 할당을 용이하게 하는지 검증합니다.

--
